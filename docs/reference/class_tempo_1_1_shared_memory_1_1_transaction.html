Detailed Description

Represents a transaction. A transaction contains one or more participants, any number of which may be locked at any one time. Participants must be locked in increasing order of id to prevent deadlock. In other words, for any locked participant p in the transaction, there does not exist an unlocked participant u where u.id < p.id
